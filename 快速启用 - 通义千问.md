# 🚀 快速启用 - 通义千问 AI 审查

## 3 步搞定

### 步骤 1：获取阿里云 API Key

1. 访问 https://dashscope.console.aliyun.com
2. 登录阿里云账号（没有就注册一个）
3. 开通 **DashScope 灵积模型服务**（免费开通）
4. 进入 **API Key 管理** 页面
5. 点击 **创建新的 API Key**
6. 复制 Key（形如 `sk-xxxxxxxxxxxxxxxx`）

---

### 步骤 2：添加到 GitHub Secrets

1. 打开 https://github.com/yuzhangCZ/agent-plugin/settings/secrets/actions
2. 点击 **New repository secret**
3. 填写：

| Name | Value |
|------|-------|
| `DASHSCOPE_API_KEY` | 你刚才复制的 `sk-xxxxx` |

4. 点击 **Add secret**

---

### 步骤 3：完成！

配置已推送到仓库，**无需额外操作**。

下次创建 PR 时，AI 会自动用通义千问审查代码！

---

## 📊 模型说明

当前配置使用 **qwen-max**（通义千问 Max 版）

| 模型 | 特点 | 价格 |
|------|------|------|
| `qwen-max` | 最强综合性能 | ¥2-4/1M tokens |
| `qwen-coder-plus` | 代码专项优化 | ¥1-2/1M tokens |
| `qwen-plus` | 性价比平衡 | ¥0.5-1/1M tokens |

**想换模型？** 编辑 `.github/workflows/ai-pr-review.yml`：
```yaml
REVIEW_MODEL: 'qwen-coder-plus'  # 改成这个
```

## ⚠️ 常见问题

**Q: API Key 报错怎么办？**
A: 检查 Key 是否正确复制，确保没有多余空格。

**Q: 审查没触发？**
A: 检查 GitHub Actions 是否运行，查看 workflow 日志。

**Q: 想换回 DeepSeek 怎么办？**
A: 改 workflow 里的 `REVIEW_MODEL: 'deepseek-chat'`，并添加 `DEEPSEEK_API_KEY` Secret。
